Developers

CSS Architecture

  1. The CSS foundation is build with a Sass preprocessor language.
  2. Icon fonts are prefixed with .cux (.cux-check).
  3. Uses modular CSS for scalable, modular, and flexible code.
  4. Uses nesting when appropriate. Nest minimally with up to three levels of nesting.
  5. Global variables are defined in the _variables.scss file. This is where custom theming can be done to change site-wide colors, fonts, etc.
  6. Media queries are built mobile first.
  7. Spacing units are as much as possible defined as rem or em units so they scale appropriately with text size. Pixels can be used for detail work and should not exceed 5px (For example: 3px borders).

CSS Best Practices

Naming

  • Avoid Camelcase
  • Write things semantically. Name it's function not its appearance.
  • Avoid presentation -or- location specific names
  • Avoid naming based on content
  • Name single noun
  • Name modifiers and state-based with adjectives
  • Consider name spacing everything

Coding

  • Do not nest more than 3 levels deep
  • Don't use IDs, IDs are reserved for JS
  • BEM (Block, Element, Modifier)
    • .accordion-item-selected
    • .nav_bar-link-clicked
  • Where possible limit files with to 80 characters

    How to set sublime to 80 characters: "rulers": [80]

  • Soft tabs with 2 space indent
  • One space after :
  • Put space before {
  • Blank line between slector block
  • Multiple slectors should be on invididual line with no space after comma
  • Property value pairs indent one level
  • Single quote urs and string values
  • Sorting (alphabetical order)
    • variables
    • extend directives
    • include directives
    • properties
  • Use rem units with px fallback

    @mixin font-size($sizevalue: 1.6){
      font-size: ($sizevalue * 10) + px;
      font-size: ($sizevalue + rem;
    }

  • Set HTML to 10px to ensure 1rem = 1px
  • Use em for positioning

Where Things Are:

  • HTML markup is located _____
  • Sass styles are located in /sass. Compiled CSS is located in /css
  • JS is located in /js/src/. Compiled JS is located in /js/build/
  • If Fonts are local then are located in /fonts, otherwise loaded from ___
  • Images are located in /images

Browser Support

The standards are designed to support both older and newer browsers through progressive enhancement, and officially support Internet Explorer 9 and up, and the latest versions of Chrome, Firefox, and Safari. Internet Explorer 8 and 9 uses polyfill.

Icon Font

Font Awesome v4.5.0

You can place Font Awesome icons just about anywhere using the CSS Prefix cux and the icon's name. Font Awesome is designed to be used with inline elements (we like the <i> tag for brevity, but using a <span> is more semantically correct).

Please note the sufix change to cux. This is to avoid our icon library from possible embeded libraries.

<i class="cux cux-camera-retro"></i> cux-camera-retro

For more information please read Font Awsesome documentation
