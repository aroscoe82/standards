Developers

  • Avoid Camelcase
  • Write things semantically. Name it's function not its appearance.
  • Avoid presentation -or- location specific names
  • Avoid naming based on content
  • Name single noun
  • Name modifiers and state-based with adjectives
  • Consider name spacing everything
  • Do not nest more than 3 levels deep
  • Don't use IDs, IDs are reserved for JS
  • BEM (Block, Element, Modifier)
    • .accordion-item-selected
    • .nav_bar-link-clicked
  • Where possible limit files with to 80 characters

    How to set sublime to 80 characters: "rulers": [80]

  • Soft tabs with 2 space indent
  • One space after :
  • Put space before {
  • Blank line between slector block
  • Multiple slectors shoud lbe on invididual line with no space after comma
  • Property value pairs indent one level
  • single quote urs and string values
  • Sorting (alphabetical order)
    • variables
    • extend directives
    • include directives
    • properties

Designers

  • Rem units with px fallback

    @mixin font-size($sizevalue: 1.6){
      font-size: ($sizevalue * 10) + px;
      font-size: ($sizevalue + rem;
    }

  • Set HTML to 10px to ensure 1rem = 1px
  • Use em for positioning
